review code 主要看哪些？

2021-10-27 1 min read # 项目管理

1.我们如何做好代码review呢 ？

前提

  1. 代码改动范围清单

基础构建配置review

  1. 有没有使用新的技术栈（符合要求的）
  2. 有没有加入本季度要求的监控
  3. 有没有线上新增的配置 拼写错误
  4. 公共代码
  5. 接口
  6. 代码封装程度

###业务代码

  1. 有没有该抽取组件的地方
  2. 有没有必要的注释
  3. 有没有对readme 进行改进
