大屏复盘

2020年5月份针对大屏做了一个新版本, 做一次分享作为复盘

5月27日晚上确定方案, 制定好计划, 并安排好人员, 拆分任务正式启动了”最美数据大屏”项目

  • 做一版 PPT(v0.1 MVP, 重点在于可交付)
  • 做一版 datav 的实现(v1.0, 重点在于简单动画)
  • 做一版代码实现(v2.0, 重点在于实现流畅自然的动画效果)

项目过程为: 7个人, 花了6天的时间, 做了7屏8页, 发了9个版本

期间遇到的最大挑战有

  1. 外部因素
  2. 浏览器崩溃
  3. 动画卡顿
  4. 翻页笔的兼容性问题

经历过大屏项目之后, 我们了解到大屏的本质: 一个长时间不间断运行的网页
由于其需要长期可用, 所以会遇到一系列平时不太需要考虑的问题, 主要有

  1. CPU 问题
  2. 内存问题
  3. 卡顿问题
  4. 崩溃问题
  5. 在长时间运行的过程中各种可能出现的意外情况

意识到大屏的要素: 内容和稳定永远是第一位的

针对此次项目的经验有

  • 别炫技, 别炫技, 别炫技
  • 逻辑越简单越好, 动画越少越好, 不出问题才是真理
  • 拒绝 iframe
  • 尽量选用 CSS 动画, 少用 JS 动画
  • 尽量选用 RAF 实现动画, 少用 setTimeout 类
  • 尽早做页面的压力测试(记得断网), 发现性能问题
  • 尽早做断网测试, 做好兜底逻辑
  • 一定要做好双机备份, 最好准备一个网页截图 PPT 以防万一

针对此次项目还能够改进的有

  • 大屏适配, 实时数据推送
  • 如果现场出现意外情况怎么办? 远程控制, 快速恢复!
  • 如何彻底避免翻页笔硬件带来的问题? 软遥控器!
  • 如何避免浏览器失去焦点?
  • 自动切换自动翻页模式与手动翻页模式
  • 用 PPT 可以做出来吗?
  • 应急预案, 万一出现系统弹窗怎么办? 主屏副屏能 hold 住所有的情况吗? 有必要做一个独立的桌面应用!
